## Senior Software Engineer - CMS, Back-office & Internal Platforms

I build and maintain long-living internal platforms - CMS, back-office systems, editorial tools and publication workflows - that organize data, automate processes, and keep business and editorial teams productive. Working in software since 2008, mostly on complex, long-lived systems where uptime and data integrity matter more than chasing the newest framework.

My focus is connecting legacy and modern technology without disrupting operations: reducing regression risk, tightening permissions, and simplifying systems instead of adding complexity.

### What I work on

- CMS and back-office platforms, editorial tools and publication workflows
- ACL / RBAC permission models and role-based interfaces
- API integrations between internal systems
- Event instrumentation and analytics
- Legacy modernization and migrations, monorepo and CI/CD setups

### How I approach it

- Modernize legacy code without breaking what already runs in production
- Treat regression risk as a first-class concern: backward compatibility and careful, incremental change
- Prefer simpler systems over unnecessary complexity
- Evidence-based decisions, backed by what the code and its history actually show
